Changelog

v1.4.2 — July 2025

  • Guided sidebar flow — Cluster Size opens by default; each section has a Next button that collapses the current section and opens the next, creating a wizard-style configuration flow
  • Config summary chips — collapsed sections show a compact summary of configured values (e.g. "512 GPUs · 64 nodes", "Two-tier · Training") so you can see your selections at a glance
  • Frontend topology packing — hosts now correctly fan into frontend leaves based on port capacity (e.g. 6 hosts per 48-port leaf at 8 links/host) instead of 1:1 round-robin
  • Backend/Frontend divider — dashed separator line with BACKEND/FRONTEND labels in the topology view
  • Optics & Power tab — merged into a single tab with backend optics+power grouped together and frontend optics+power grouped together, so the LPO toggle immediately reflects in power figures
  • Host ellipsis count — host row ellipsis in topology now shows omitted count (e.g. "…+12 more") consistent with other tiers
  • Frontend topology tooltip fix — frontend leaf connection counts now use actual host totals instead of displayed node count

v1.4.1 — July 2025

  • Spine balance fix — Single-Plane vs Multi-Plane comparison cards now apply the same balance adjustment as the main calculation, fixing mismatched spine counts
  • B300 GPU defaults — NVIDIA B300 now correctly defaults to 2× ConnectX-7 at 400G instead of 1× ConnectX-8 at 800G

v1.4 — July 2025

  • Tabbed results pane — results organized into Summary, Fabric, Optics, and Power tabs to reduce clutter; selected tab persists across recalculations
  • NIC breakout topology fix — multi-plane rail view now correctly shows all GPUs connecting to every plane (e.g. 8×200G per plane instead of 4×200G); NIC expand animation updated to match
  • Spine optics for breakout — spine-facing optics now show OSFP 800G 2DR4 (dual MPO-12) instead of DR8 (single MPO-16) when uplink breakout is active, matching the natural connector topology for 2:1 breakout

v1.3 — July 2025

  • Modular Spine Advantage — side-by-side comparison of modular 7800R4 chassis vs all-fixed designs showing platforms, rack space (RU), optics, and power with per-row highlighting and recommendation notes
  • GPU / NIC selection — choose from NVIDIA (GB300, GB200, B300, B200, H200, H100) and AMD (MI350X, MI325X, MI300X) with auto-configured NIC and link speed
  • Training / Inference toggle — workload-aware guidance for fabric design (E-W vs N-S traffic patterns)
  • FRO / LPO optics — renamed from Conventional, added annual OpEx savings estimate and CapEx guidance for 800G modules
  • Flow visualization — click two compute nodes to trace traffic path with animated packets and device count classification (intra-rail, inter-rail, intra-pod, inter-pod)
  • Topology export — PNG and SVG via Export dropdown in the topology viewer
  • Topology redesign — Fat Tree / Rail pill toggle replaces dropdown; multi-plane always shows all planes with correct rail-optimized GPU-to-plane mapping (GPUs 0–3 → Plane 1, 4–7 → Plane 2)
  • GPU/NIC expand — double-click a host to see per-GPU leaf connections inline (no overlay); lines draw to real leaf positions
  • Spine-link breakout — host breakout auto-applies to spine links, fixing 8192-GPU 2×400G multi-plane topologies
  • 3-tier naming — renamed to Super-spine / Spine / Leaf with connectivity metrics (full/partial mesh)
  • 400G LPO removed — LPO only shown for 800G optics (the only speed where it exists)
  • Multi-plane naming — renamed from "MRC Multi-Plane" to "Multi-Plane"
  • NIC breakout toggle — Distribute / NIC Breakout pill for multi-plane with fewer NICs than planes
  • Leaf platform fix — 7800R4 chassis no longer suggested as leaf switches
  • Changelog — version history accessible from the top bar

v1.2 — June 2025

  • Custom CSS tooltips with info icons
  • 3-tier pod-based Clos with proportional spine-agg connectivity
  • Rail-optimized topology view with per-GPU leaf mapping
  • MRC multi-plane topology view with plane-colored backgrounds
  • Click-to-expand GPU/NIC detail in topology

  • Power consumption estimates (Typical / 80% / Max)
